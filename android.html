Tauri Masterclass: Build & Sign Android Apps

Take your web application and compile it into a fully native Android APK. We will cover the "True Dev" method of manually aligning and signing the APK from the terminal to avoid Android Studio bugs.

1. Android Prerequisites

Before adding Android to your Tauri project, you need the mobile SDKs and Rust targets.

  1. Install Android Studio (Ensure SDK and NDK are installed via SDK Manager).
  2. Add Rust Mobile Targets by running this in your terminal:
rustup target add aarch64-linux-android armv7-linux-androideabi i686-linux-android x86_64-linux-android

2. Initialize & Test on Phone

Navigate to your existing Tauri project folder. Inject the Android native files by running:

npm run tauri android init

To test the app live on your physical Android phone, plug it in via USB (ensure USB Debugging is ON) and run:

npm run tauri android dev

3. Build the Unsigned Release

To generate the highly optimized APK file for production, run the build command:

npm run tauri android build
Result: Your raw, unsigned APK is located at: src-tauri/gen/android/app/build/outputs/apk/universal/release/app-universal-release-unsigned.apk

4. Manual Signing Process (The True Dev Way)

Modern Android phones (Android 11+) reject basic signatures. We must use the official SDK tools (zipalign and apksigner) to apply a strict V2 Signature. (Replace paths below with your actual Windows username/paths).

Step 4.1: Generate a Keystore (One-Time Only)

Run this tool (which comes with Android Studio's JBR) to create your digital identity. Note the C=TZ stands for Tanzania!

"C:\Program Files\Android\Android Studio\jbr\bin\keytool" -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias -storepass 123456 -keypass 123456 -dname "CN=Dev, OU=Personal, O=Personal, L=Unknown, ST=Unknown, C=TZ"

Step 4.2: Set JAVA_HOME

The signing tools require Java. Point your terminal to Android Studio's bundled Java:

set JAVA_HOME=C:\Program Files\Android\Android Studio\jbr

Step 4.3: Zipalign the APK

This rearranges the APK's memory footprint so Android can read it efficiently. (Make sure you change the path to your actual SDK build-tools folder version).

"C:\Users\YOUR_USERNAME\AppData\Local\Android\Sdk\build-tools\34.0.0\zipalign.exe" -f -v -p 4 "src-tauri\gen\android\app\build\outputs\apk\universal\release\app-universal-release-unsigned.apk" app-aligned.apk

Step 4.4: Sign the APK

Finally, apply the V2 signature using apksigner. This creates your final, shareable app!

"C:\Users\YOUR_USERNAME\AppData\Local\Android\Sdk\build-tools\34.0.0\apksigner.bat" sign --ks my-release-key.jks --ks-key-alias my-key-alias --ks-pass pass:123456 --key-pass pass:123456 --out my-final-app.apk app-aligned.apk
Success! You now have a file named my-final-app.apk in your project folder. Transfer it to your phone and install it flawlessly!
